Dùng trang này như checklist triển khai: cài công cụ, kết nối trong VS Code, và thao tác cơ bản.

Nhóm SQL

PostgreSQL
Cài: installer hoặc Docker. Dùng nhanh: tạo database, user, mở psql hoặc GUI để query.
Cài đặt Docs
MySQL / MariaDB
Cài: MySQL Installer hoặc MariaDB Installer. Dùng nhanh: tạo schema, cấp quyền user ứng dụng.
MySQL MariaDB
SQL Server
Cài: SQL Server Developer + SSMS. Dùng nhanh: tạo login, database, kiểm tra bằng truy vấn SELECT.
Cài đặt Docs
YugabyteDB / CockroachDB / TiDB
Cài: chạy local cluster hoặc cloud trial. Dùng nhanh: kiểm tra tương thích SQL và latency đa vùng.
YugabyteDB CockroachDB TiDB

Nhóm NoSQL

MongoDB
Cài: MongoDB Community. Dùng nhanh: tạo collection, index, chạy truy vấn filter/sort.
Cài đặt Docs
Redis
Cài: bản phù hợp hệ điều hành hoặc chạy container. Dùng nhanh: set/get key, TTL, cache API.
Cài đặt Docs
Cassandra / ScyllaDB
Cài: single-node để thử nghiệm trước. Dùng nhanh: tạo keyspace, table theo query pattern.
Cassandra ScyllaDB
Firebase
Cài: dùng Firebase Console + Firebase CLI. Dùng nhanh: tạo project, bật Firestore/Auth, cấu hình SDK app.
Docs Firebase CLI
Supabase
Cài: tạo project trên cloud hoặc chạy local bằng Supabase CLI. Dùng nhanh: tạo bảng, policy RLS, kết nối client SDK.
Local Dev Docs

Extension VS Code tương thích

Công cụ Extension gợi ý Cách dùng nhanh trong VS Code
PostgreSQL SQLTools + PostgreSQL Driver, hoặc PostgreSQL extension Tạo connection, chạy file .sql, lưu snippet query
MySQL / MariaDB SQLTools + MySQL/MariaDB Driver Tạo profile kết nối, browse schema, chạy query
SQL Server SQL Server (mssql) Sign-in nếu cần, mở query editor, chạy truy vấn
MongoDB MongoDB for VS Code Connect cluster/local, chạy playground, xem schema
Redis Redis for VS Code Kết nối endpoint, duyệt keyspace, test command
Neo4j Neo4j for VS Code Kết nối graph DB, chạy Cypher, trực quan kết quả
Firebase Firebase VS Code Extension Quản lý project, rules, functions và deploy từ VS Code
Supabase Supabase VS Code Extension + SQLTools Quản lý project Supabase và truy vấn PostgreSQL trực tiếp
Đa hệ CSDL Database Client JDBC hoặc SQLTools Dùng một giao diện để quản lý nhiều loại database

Quy trình chuẩn đề xuất

1. Cài DB cục bộ

Cài bản local hoặc chạy Docker để tạo môi trường dev nhanh.

2. Cài extension

Cài extension theo loại DB đang dùng để kết nối trực tiếp từ VS Code.

3. Tạo migration

Dùng Flyway, Liquibase, Alembic hoặc Prisma Migrate ngay từ đầu.

4. Bật monitoring

Thêm dashboard và cảnh báo sớm trước khi lên production.

Nếu cần, bạn có thể chuẩn hóa thành checklist onboarding cho toàn team.
